# Single Page Application

## 싱글 페이지 애플리케이션이란?

**렌더링과 라우팅에 필요한 대부분의 기능을 서버가아닌 브라우저의 자바스크립트에 의존하는 방식을 의미**

- 최초 첫 페이지에서 데이터를 모두 불러온 후에 페이지 전환을 위한 모든 작업이 자바스크립트와 브라우저의 history API에 의해 이뤄지므로 페이지를 불러온 이후 서버에서 `HTML`을 내려받지 않고 하나의 페이지에서 모든 작업을 처리하게 된다.

- 사이트 렌더링에 필요한 `HTML`의 `body` 부분을 자바스크립트 코드를 삽입한 이후 렌더링한다.
- 페이지 전환시에도 새로운 HTML 페이지를 요청하는것이나닌, 자바스크립트에서다음 페이지의 렌더링에 필요한 정보만 HTTP 요청을 통해 가져온다음, 그 결과를 바탕으로 `<body/>` 내부에 DOM을 추가,수정,삭제하는 방법등으로 페이지의 전환이 이루어진다.
  - 이러한 방식은 최초 로딩시 자바스크립트 리소스가 커지지만, 한번 로딩된 이후 서버를 거쳐 필요한 리소스를 가져올 필요가 없으므로 이후 페이지 전환은 서버사이드 렌더링보다 비교적 빠르게 진행될 수 있다.
